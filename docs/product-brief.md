# `jarsplit` — продуктовий бриф

## Що це

`jarsplit` — невеликий детермінований CLI на Go, який автоматизує щомісячний розподіл коштів по особистих банках (jars) monobank.

На вхід — простий план фіксованих сум по банках. На вихід — список готових посилань `send.monobank.ua` із **уже підставленою сумою** (`?a=`), які лишається лише відкрити й підтвердити.

Назви банок у плані резолвляться в їхні `sendId` **наживо через персональний API monobank** (`GET /personal/client-info`), авторизація — токеном з env-змінної `MONO_TOKEN` (на диск не пишеться). Ендпоінт має ліміт (~1 запит/60с), тож на запуск робиться один виклик.

**Визначальна властивість — без податків і комісій.** Інструмент працює виключно з **власними особистими банками** користувача й поповненням із **власного рахунку** monobank. Він ніколи не генерує посилання на банки ФОП/бізнес-рахунків і не передбачає переказів з карток інших банків. Тому кожне згенероване посилання — це внутрішнє поповнення власної банки: без податку й без комісії.

Свідомо поза скоупом: відсотки та «решта» — **лише фіксовані суми**.

## Для кого

Це особистий інструмент автора — користувача monobank, який щомісяця **вручну розкидає кошти по власних банках** (заощадження, подушка, подорожі тощо) і хоче зробити це за один запуск замість десятка ручних дій.

Це не масовий продукт: один користувач, один рахунок, власні банки. Простота й чесність скоупу тут навмисні.

## Який біль вирішує

Щомісячний розподіл вручну — це рутина, яка займає час і легко ламається:

- відкрити застосунок, знайти кожну банку окремо;
- згадати, скільки саме йде в цю банку цього місяця;
- ввести суму вручну — і повторити для кожної банки;
- легко помилитися: не та сума, не та банка, забута банка.

`jarsplit` зводить це до **одного запуску й кількох кліків**: ти описуєш план один раз, отримуєш точні посилання з підставленими сумами та **друкований підсумок (total)** — швидку перевірку, що сумарно розподіляється рівно стільки, скільки планувалося.

## Наскрізний сценарій

**1. Токен** (один раз на сесію):

```
export MONO_TOKEN=...   # персональний токен monobank
```

**2. План** — звичайний текстовий файл `plan.txt`, по одній банці на рядок, суми у гривнях:

```
Заощадження = 5000
Подорожі    = 3000
Подушка     = 1500
```

**3. Запуск:**

```
jarsplit plan.txt          # або: cat plan.txt | jarsplit
```

**4. Вивід** — таблиця у stdout: банка, сума, готове посилання, і підсумок:

```
Банка          Сума       Посилання
Заощадження    5000 ₴     https://send.monobank.ua/jar/4xR…?a=5000
Подорожі       3000 ₴     https://send.monobank.ua/jar/8kP…?a=3000
Подушка        1500 ₴     https://send.monobank.ua/jar/2wQ…?a=1500
Разом          9500 ₴
```

Клікаєш кожне посилання в терміналі й підтверджуєш платіж. Опційно:

```
jarsplit plan.txt --open   # відкрити всі посилання в браузері одразу
```

**Невідома назва банки → skip + warn.** Якщо банки з плану немає на рахунку (або назва збігається неоднозначно), `jarsplit` **не зупиняється і нічого не перенаправляє мовчки**: обробляє ті банки, що збіглися, друкує для них посилання, а про незбіглі виводить попередження зі списком доступних назв. Гроші ніколи не йдуть «навздогад».

---

### Припущення (відкриті питання для реалізації)

- **Одиниці `?a=`:** треба підтвердити, чи параметр приймає гривні (`5000`), чи копійки (`500000`) — і конвертувати відповідно.
- **Поля `client-info`:** очікувані поля банки — `title`, `sendId`, `balance`, `goal`; суми в API — у копійках (×100). Точну схему верифікувати на етапі реалізації.
